Conduită & Workflow pentru Dezvoltare Proiecte

Editabil • Iunie 2025

Principii cheie

  • Claritate înainte de viteză, livrare incrementală (≤ 2 zile/task).
  • Totul documentat în repo (/docs).
  • Definition of Ready/Done obligatorii.
  • Automatizare pentru calitate (lint, teste, CI/CD).
  • Securitate și observabilitate implicite.
  • Ownership clar pe task.

Definition of Ready

  • Scop și outcome clare.
  • Criterii de acceptare scrise.
  • Design minim (flow/mocks).
  • Dependențe și riscuri identificate.
  • Estimare ≤ 2 zile.
  • Plan rollout & rollback.

Definition of Done

  • Cod în main, PR aprobat.
  • CI verde (build, lint, teste).
  • Loguri/metrics adăugate.
  • Docs actualizate.
  • Feature flag + rollback testat.

Workflow

  • Intake & triere (issues).
  • One‑pager (PRD scurt).
  • Tech spec.
  • Implementare (branch scurt, commits clare).
  • Code review.
  • Testare (unit/integration/QA).
  • Release (feature flag, canary).
  • Observare & rollback dacă e nevoie.
  • Post‑mortem fără vinovați.

Ritualuri

  • Daily async (Ieri/Azi/Blocaje).
  • Planificare săptămânală (DoR gata).
  • Retro bilunar (3 acțiuni concrete).

Git & CI/CD

  • Trunk‑based simplu (feat/, fix/, chore/).
  • Conventional commits.
  • SemVer + changelog automat.

Securitate

  • Fără secrete în repo.
  • Dependențe pinned, scan SCA.
  • Input validat, least privilege.
  • Date personale minimizate.

Templatede

  • One‑pager (Problemă, Scop, KPI, Riscuri, Rollout).
  • Tech spec (arhitectură, API, securitate, testare, rollout).
  • PR checklist (teste, lint, docs, observabilitate).
  • Release checklist (CI verde, flag, monitorizare, runbook).
